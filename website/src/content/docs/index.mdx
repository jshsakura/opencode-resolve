`opencode-resolve` adds a focused `resolver -> coder` loop to OpenCode. It keeps the default path simple, lets read-only subagents handle discovery and review when useful, and avoids turning your OpenCode config into a large prompt dump.

<div class="card-grid">
  <div class="doc-card">
    <h3>Install once</h3>
    <p>Register the plugin, refresh OpenCode's plugin cache, and keep resolve-specific settings in <code>resolve.json</code>.</p>
  </div>
  <div class="doc-card">
    <h3>Use small roles</h3>
    <p><code>resolver</code> orchestrates. <code>coder</code> edits. Scout and review agents stay read-only by default.</p>
  </div>
  <div class="doc-card">
    <h3>Pin models only when needed</h3>
    <p>By default, all resolve agents inherit OpenCode's top-level model. Add tiers only when they help.</p>

## Fast Path

```sh
npm install -g opencode-resolve
opencode plugin opencode-resolve --global --force
opencode
```

Then select `resolver` in OpenCode.
